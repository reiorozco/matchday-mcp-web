# Product

## Register

brand

## Users
Two audiences land here from a GitHub README, an npm page, or a LinkedIn "Featured" card:
1. **Hiring engineers / tech recruiters** skimming a portfolio between meetings — they want fast signal that the author can design and build production-grade software (Svelte 5, TypeScript, MCP, clean APIs).
2. **Developers & football fans** who want to understand what the `matchday-mcp` server does and see it working with real data before installing it.

## Product Purpose
The live playground for `matchday-mcp`, an open-source Model Context Protocol server that gives AI assistants live football data. The page must (a) explain what the MCP server is and how to install it, and (b) prove it works with a real, interactive dashboard (standings, team form, head-to-head comparison, top scorers) powered by the same data layer the server uses. Success = a visitor leaves impressed by the craft and clear on what the project is.

## Brand Personality
Confident, energetic, precise. The voice of a modern sports broadcast graphics package crossed with a well-built developer tool — "matchday floodlights" energy, but every number is exact and every interaction is fast. Three words: **sharp, kinetic, credible.**

## Anti-references
- Grass-green "football website" clichés (pitch textures, turf gradients, whistle icons).
- ESPN / Sky Sports dark-neon stats-terminal look (the second-order sports-data reflex).
- Generic SaaS-cream landing with a hero-metric template and identical icon-card grids.
- Editorial-magazine affectation (display-serif italic + drop caps) — wrong register here.

## Design Principles
- **Show, don't tell.** The dashboard runs on the same client as the MCP server; the proof is the working product, not a screenshot.
- **Broadcast energy, terminal precision.** Bold, wide display type and matchday drama up top; exact tabular data and fast interactions below.
- **Two worlds, one voice.** A dark, floodlit hero gives way to a bright, legible dashboard — art-directed per section, consistent in voice.
- **Every figure is real and legible.** Tabular numerals, verified contrast, honest data straight from the API.

## Accessibility & Inclusion
WCAG AA: body text ≥4.5:1, large text ≥3:1, visible focus states, keyboard-operable tabs and inputs. W/D/L is never encoded by color alone (always a letter too). Full `prefers-reduced-motion` alternative for every animation.
